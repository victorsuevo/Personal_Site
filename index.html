Created a scheduling control system for a pediatric clinic

The scheduling control system developed in collaboration with the Burtolini Pediatric Clinic includes several essential features designed to streamline appointment management and enhance patient care. The code, written in Python with SQLite as the database, is structured to address the clinic's specific needs effectively. Here's an overview of its features and functionality:
1. Appointment Scheduling: The system enables precise scheduling of appointments, preventing overlapping time slots. It supports emergency bookings that bypass the standard 30-minute interval restriction, ensuring flexibility during critical situations.
2. Patient Registration: New patients can be registered effortlessly while avoiding data duplication, such as duplicate CPF entries. This ensures accurate and organized patient records.
3. Edit and Cancel Appointments: Users can edit, cancel, or reschedule appointments directly through the interface, offering adaptability for last-minute changes.
4. Time Interval Control: For non-emergency cases, the system enforces a 30-minute gap between appointments. This rule is overridden for emergency bookings to prioritize urgent care.
5. Appointment Display: A user-friendly interface displays scheduled appointments clearly, with options to edit or cancel directly from the list.
6. Report Generation: The system can generate CSV reports for administrative purposes, such as financial tracking and patient flow analysis.

Implementation:

- Database Management with SQLite: The system uses SQLite for local data storage, handling patient and appointment information. Its simplicity eliminates the need for a complex database server while ensuring reliable data management.
- Table Structure: - Patients: Stores patient details like name, CPF, and age, with validation to prevent duplicate records.
- Appointments: Records appointment details, including date, time, patient, and type (regular or emergency), ensuring no schedule conflicts.
- Graphical User Interface (GUI): Built using Tkinter, the interface is designed for simplicity and ease of use. Key components include: - Buttons for scheduling, editing, and canceling appointments, placed intuitively for quick access. - A Listbox to display scheduled appointments, allowing edits or cancellations directly. - Input fields for patient and appointment details, with validations to prevent errors or redundancies.
- Validation and Scheduling Logic: The code validates appointment timings, ensuring they comply with interval rules unless marked as an emergency. Any scheduling conflicts are flagged for user correction.
- Editing and Canceling Appointments: Users can seamlessly modify appointment details or remove bookings when necessary, streamlining administrative tasks.
- Report Export Functionality: CSV report generation simplifies administrative workflows by providing insights into patient flow, financial data, and operational efficiency.
- Scalability and Maintenance: Designed with adaptability in mind, the code can accommodate future enhancements, such as system integration, automated reminders, or expansion to multiple clinic branches.
This system is designed to optimize the clinicâ€™s workflow, reduce human error, and ensure a more organized and patient-focused environment. It allows the administrative and medical staff to concentrate on delivering quality care rather than dealing with logistical challenges.
